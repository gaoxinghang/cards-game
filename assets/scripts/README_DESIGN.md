# 纸牌 Demo 程序设计

## 架构

工程按数据、规则、控制器、视图、可撤回动作分层：

- `models/CardTypes.ts`：定义卡牌点数、花色、区域和坐标等数据。
- `models/GameState.ts`：维护当前顶部牌、全部卡牌和撤回历史栈。
- `rules/CardRule.ts`：封装“点数差 1 即可匹配”的规则。
- `controllers/GameController.ts`：处理点击、匹配、移动和撤回入口。
- `actions/IUndoableAction.ts` 与 `actions/MoveCardAction.ts`：统一动作执行和撤销。
- `views/CardView.ts`：负责卡牌显示、点击和移动动画。
- `CardGameScene.ts`：运行时生成关卡、区域背景、卡牌、撤回按钮和下一关按钮。

## 当前玩法

项目没有预置场景文件。打开 Cocos Creator 3.8.8 后，新建一个空场景，创建 `Canvas` 节点并挂载 `CardGameScene` 组件即可运行。

主牌区卡牌必须与顶部牌点数相差 1 才能移动到顶部牌位置。被其他主牌压住的卡牌显示为背面，不能点击；上层牌移走后，下面的牌会自动翻开。手牌 / 备用牌区的卡牌可以直接移动到顶部，用来衔接断开的牌面。点击撤回按钮可以回退最近一步。

当前包含 3 个关卡，通关路线分别是：

- 第 1 关：`3 -> 2 -> 7 -> 8`
- 第 2 关：`5 -> 4 -> 8 -> 9 -> 10`
- 第 3 关：`6 -> 5 -> 4 -> 9 -> 10 -> 11 -> 12`

清空当前关卡的主牌区后，会显示通关提示。前两关会出现“下一关”按钮，点击后进入下一关；第三关完成后显示“全部通关”。

## 扩展方式

新增关卡时，在 `CardGameScene.levels` 数组中追加 `LevelConfig` 配置即可。每关配置包含顶部牌、主牌区卡牌、备用牌区卡牌、点数、花色、区域和初始坐标。主牌区可通过 `stackId` 和 `stackIndex` 设置叠放关系，同一个 `stackId` 内 `stackIndex` 最大的牌会正面朝上，其余被遮挡的牌显示背面。

新增撤回类型时，实现 `IUndoableAction` 的 `execute()` 和 `undo()`，在对应业务逻辑执行后压入 `GameState.history`。例如翻牌、发牌、洗牌、多牌连消都可以复用同一个历史栈。

## 注意事项

动画期间 `GameState.inputLocked` 会锁住输入，避免重复点击导致状态错乱。花色只用于显示，不参与匹配判断。
